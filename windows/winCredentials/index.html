如题

问题现象

  1. 一开始同事将 git 安装到自定义的目录，然后安装的 tortoisegit，然后用 tortoisegit 的界面 clone 项目，出现登录提示框，输入用户名密码之后，提示 git.exe 找不到。然后同事又忘了将 git 安装到哪个目录了，悲催呀，然后我建议他全部重新安装，都安装到默认目录，并且重启电脑。
  2. 如此操作之后，找不到 git.exe 的问题没有了，但是认证失败依旧。彻底卸载之后，重启电脑，再重新安装，还是不行。确定 git 账号和权限都是可以的，在另外的电脑上操作，一切正常。
  3. 删除 tortoisegit 的认证数据也不行，其实这一步多余，因为都卸载了重装了的。

win7 解决

win7 最终解决办法：进入控制面板->用户账户->用户凭据，删除 gitee 的凭据，然后再试就 OK 了。

win 10 用户凭据管理

删除登录凭据的操作如下（二选一）：

  1. win key+R，输入“control userpasswords2”，打开【用户账户】-> 高级-> 管理密码->windows 凭据
  2. 进入【控制面板】->用户账户->管理 Windows 凭据
    通过以上方式打开如下界面，点开【普通凭据】，找到 gitee 的记录，删除。
