如题

编辑 conf/server.xml
将 localhost 的 Host 配置项替换成下面的。这样操作以后，tomcat 的默认页面不可访问，不过，一般的生产环境和开发环境也不需要自带的管理页面。

可以配置多个 Host。但 name 不能重复。
配置为 localhost 的 Host 可以直接用 IP + 端口的方式访问。这种方式也可以配置域名访问，只要域名指向服务器 IP 即可。
如果 name 是一个域名，则只能用域名访问（域名必须指向该服务器）。

网上的其它方案：在/conf/Catalina目录下建立目录“您要配置的域名的名称”,然后建立ROOT.xml文件，其格式和/conf/localhost目录下的ROOT.xml一致…
貌似已经不适用于 tomcat-8.5 以后的版本了。大概是版本 6 的配置。
